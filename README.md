# Portfolio Backend

NestJS + TypeScript + PostgreSQL (managed by dbmate) + Kysely + Sanity + Resend + Firebase Admin + AWS S3.

## Quick start

```bash
cp .env.example .env       # fill in secrets (dbmate + NestJS both read this)
docker compose up -d       # local Postgres on :5432 + Redis on :6379
npm install
npm run db:up              # run dbmate migrations (uses DATABASE_URL from .env)
npm run db:codegen         # regenerate src/database/types.ts from schema
npm run start:dev          # nest start --watch on :3001
curl http://localhost:3001/health
```

## Scripts

| Script | Purpose |
| --- | --- |
| `npm run start:dev` | Dev server with watch |
| `npm run build` | Compile TS → `dist/` |
| `npm run start:prod` | `node dist/main.js` |
| `npm run db:up` | Apply pending dbmate migrations |
| `npm run db:down` | Rollback latest migration |
| `npm run db:status` | Show migration state |
| `npm run db:new <name>` | Generate a new migration file |
| `npm run db:codegen` | Regenerate Kysely types from live schema |

## Layout

```
src/
├── main.ts                — bootstrap (helmet, CORS, global pipes/filters)
├── app.module.ts          — root module
├── health.controller.ts   — GET /health
├── config/                — ConfigModule + zod-validated Env
├── common/                — filters, guards, decorators
├── database/              — Kysely instance + codegen'd types
├── media/                 — S3 presigned uploads
│   ├── media.controller.ts (POST /media/presign, DELETE /media/:key)
│   └── s3.service.ts
... (more modules added in subsequent phases — see plan file)
db/
├── migrations/            — dbmate SQL files (timestamped, with -- migrate:up / -- migrate:down)
└── schema.sql             — auto-generated by dbmate on every up/down
```

## Docker

Production image (Node 22 alpine, multi-stage; entrypoint runs dbmate migrations
before starting Node, so a fresh container always has the latest schema):

```bash
docker build -t portfolio-api .
docker run --env-file .env -p 3001:3001 portfolio-api
```

The container expects `DATABASE_URL`, `REDIS_URL`, all `JWT_*`, `OWNER_EMAIL`,
`SANITY_*`, `S3_*`, `RESEND_*`, `FIREBASE_*`, `CLIENT_REVALIDATE_URL`, and
`REVALIDATE_SECRET` in the env (see `.env.example`).

## Phases

This server is built in committed phases (see `~/.claude/plans/i-am-have-empty-woolly-clock.md`):

- **Phase A**: infrastructure (config, db, media) — ✅ this commit
- **Phase B**: auth (email/pass + refresh tokens + Google via Firebase)
- **Phase C**: portfolio CRUD + `GET /api/portfolio` aggregator
- **Phase D**: Sanity Studio scaffold in `../sanity-studio/`
- **Phase E**: posts mirror + comments + likes + views + suggestions + webhook
- **Phase F**: blog subscriptions + new-post dispatcher (Resend)
